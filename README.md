# PlantDiseaseDetection
Agriculture plays a pivotal role in feeding a human and livestock population.Especially in developing countries like India where Agriculture is the backbone of economy where 55% of population depends on agriculture. The occurrence of plant disease and pests is one of the major reason of  yield Reduction and crop failure.Early prediction and prevention of plants disease and pests are some of the important research areas in agricultural field. 


Effective detection of plant diseases is critical for maintaining agricultural productivity, ensuring food security, and minimizing economic losses. However, current detection methods often lack accuracy, efficiency, and scalability, posing challenges for timely intervention and sustainable crop management. Addressing these limitations is crucial to enhance disease surveillance, optimize resource use, and promote resilient agricultural practices globally.

Project setup
Step 1: Create a repository on DagsHub
I will show how I made the setup from scratch. You can create a new project on DagsHub by connecting to a project you have on GitHub. This enables both repositories to be synchronized.

When you create a repository on DagsHub you will have access to three remote servers e.g DVC, MLflow & Git, that are automatically configured with this repository.

Clone the project and set up a virtual python environment, by running the following commands:

    git clone https://github.com/whogaurab/PlantDiseaseDetection.git
    conda create -p venv python==3.11 -y
    conda activate venv/

install requirements

    pip install -r requirements.txt
